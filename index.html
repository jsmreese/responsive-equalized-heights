Responsive Equalized Heights

for ZURB Foundation

An intuitive, class-based interface for creating responsively equalized element heights within ZURB Foundation.

Maintained by jsmreese.

Why this project?

There are a few height-equalization projects in the wild, and Foundation itself includes the Equalizer component. I've found that Equalizer handles basic use-cases well, but breaks down for the responsively updating groupings I needed.

I love Foundation's class-based syntax for responsive grid column widths (e.g. medium-6, large-4) and I wanted to be able to specify responsive element height groupings in a similar, intuitive way.

Format

Responsive height class name format is <screenSize>-height-<group>.

screenSize

Any of the Foundation screen-size media query names:
small, medium, large, xlarge, or xxlarge.

group

Any arbitrary string that creates a valid class name. I most often use one or two lowercase letters, but medium-height-iCaNhAS-cHeEzBuRgEr will also work.

auto is a reserved group name and will not trigger equalized heights. Use it to override smaller screen equalization by reverting to auto-height layout on larger screens.

Basics

Responsive element height classes act just like column width classes with regard to screen size:

  • Adding a class of medium-height-a to two elements will equalize their height on medium and larger screens.
  • Adding a class of large-height-a to another element will equalize the height of all three elements on large and larger screens.
  • Adding a class of small-height-b to two elements will equalize their height for all screens.

Blocks that are in the same height group will share equal height.

Multiple height classes may be used to specify different height groups based on screen size. Using the classes medium-height-a large-height-b would match height group a on medium screens and height group b on large and larger screens.

Height classes that are not active, such as the class xlarge-height-a on a medium screen, are ignored when determining group height.
